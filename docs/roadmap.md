# Roadmap

## Phase 0: Definition

Goal: lock the concept tightly enough to build.

Deliverables:

- Product brief
- Seven dwarf roster
- First hub map sketch
- First 10 quests
- Inventory item schema
- Project model
- Spatial interface model
- Technical stack decision

Exit criteria:

- A builder can explain the game in one sentence.
- A player can understand why they should talk to each dwarf.
- V1 can be built without needing a massive open world.
- OpenClaw and Kingdom of PAL have clear responsibilities.

## Phase 1: Playable Prototype

Goal: prove the world interaction loop.

Build:

- Browser scene
- One controllable player
- One hub area
- One dwarf NPC: Orin Forgekeeper
- Conversation panel
- Project board
- One project room
- One quest
- Inventory item creation
- Attach item to project room
- Local save

Exit criteria:

- Player can create a project, walk to Orin, complete a quest, receive a saved prompt scroll, and attach it to the project room.

## Phase 2: Hackathon MVP

Goal: prove the full spatial agent workspace loop.

Build:

- Seven dwarf NPCs
- Quest log
- Inventory UI
- Skill bookshelf
- Project board
- Project room visualization
- 10 practical quests
- Agent output contracts
- Asset vault screen
- Basic export to markdown
- Lightweight onboarding

Exit criteria:

- A new player can complete 3 quests in 15 minutes and leave with useful saved assets.
- A new player can understand the status of one project by walking into its room.

## Phase 3: Private Alpha

Goal: validate retention and usefulness with Prompt Alchemist Labs members.

Build:

- Accounts
- Persistent cloud inventory
- Persistent project rooms
- Skill books and learning progression
- Player memory
- OpenClaw integration for agent runtime
- Better quest recommendations
- Weekly featured quests
- Community vault highlights
- Analytics for quest completion and repeat visits

Exit criteria:

- At least 30 percent of testers return in week 2.
- At least 50 percent of completed quests generate an exported or saved asset.
- Users can name which dwarf helped them and why.
- Users can name which project improved because of the session.

## Phase 4: Productization

Goal: turn the game into a monetizable learning and agent platform.

Build:

- Paid quest packs
- Workshop-linked events
- Team workspaces
- Team project worlds
- Public asset sharing
- Advanced agent tools
- Template marketplace
- Founder/operator learning paths

Exit criteria:

- Users pay for outcomes, not novelty.
- The world drives signups to Prompt Alchemist Labs products.
- Saved assets become reusable community IP.

## Deliberate Later

Do not build these before the core loop works:

- Large world map
- Real-time multiplayer
- Complex combat
- Player housing
- Blockchain assets
- Marketplace economy
- Procedural terrain
- Mobile native app
