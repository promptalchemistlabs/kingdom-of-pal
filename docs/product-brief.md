# Product Brief

## One-Liner

Kingdom of PAL is a RuneScape-inspired open-world spatial interface where players walk through a 2.5D/3D kingdom to talk to dwarven AI agents, manage inventory, visualize projects, and turn ideas into prompts, workflows, automations, and business assets.

## Core Promise

Enter the kingdom with raw ideas. Leave with forged assets, visible projects, and reusable workflows.

Examples of forged assets:

- Refined prompts
- Workflow templates
- Business model sketches
- Landing page copy
- Automation plans
- Learning quests
- Content outlines
- Skill books
- Project maps
- Product positioning notes
- Investment or market research checklists

## Positioning

Kingdom of PAL should feel like:

- A game world first, not a chatbot page with decorations.
- A spatial command center for agents, projects, workflows, and knowledge.
- A learning and execution hub, not passive content.
- A living interface for Prompt Alchemist Labs, not just lore.
- A practical agent RPG, not heavy fantasy roleplay.

## Relationship To OpenClaw

OpenClaw is the backend agent and workflow execution layer.

Kingdom of PAL is the embodied world interface.

```text
OpenClaw = agents, tools, workflow execution, memory
Kingdom of PAL = open-world navigation, project visualization, inventory, quests, embodied conversations
```

The world should eventually let the player talk to the same agents through place, movement, objects, and project context instead of only through a chat or OpenClaw-style control surface.

## Inspirations

Use these as references, not templates:

- RuneScape: explorable world, NPCs, inventory, skills, quests, approachable RPG systems.
- Animal Crossing: cozy repeat visits, NPC personality, collection, light progression.
- Minecraft: only the broad feeling of open-ended exploration and player agency.
- Spatial dashboards: projects and workflows represented as places, objects, rooms, and boards.
- AI agent workbenches: task delegation, memory, generated outputs, workflow execution.

## Anti-Positioning

This should not be:

- A voxel Minecraft clone.
- A massive MMO in v1.
- A generic chatbot with a 3D background.
- A game map that does not help users manage real work.
- A lore-heavy fantasy RPG where usefulness gets buried.
- A crypto/metaverse project before the core experience is proven.

## Target Users

Primary users:

- AI-curious builders
- Prompt Alchemist Labs community members
- Hackathon participants
- Indie hackers and technical founders
- Students or professionals learning applied AI

Early users should have a reason to return weekly:

- They want better prompts.
- They need help structuring ideas.
- They want small quests to practice AI.
- They want a memorable way to access templates, workflows, and agents.

## V1 Product Shape

V1 is a browser-based 2.5D/3D compact open-world district.

Minimum world:

- One playable character
- One town hub
- Seven dwarven AI agents
- One project board
- One project room
- Skill bookshelf
- Workflow forge
- Inventory panel
- Quest log
- Conversation UI
- Generated asset vault
- Basic save state

Minimum gameplay:

- Move around the hub.
- Approach a dwarf.
- Start a conversation.
- Accept a quest.
- Create or select a project.
- Give context or an idea.
- Receive a useful output.
- Save the output as an inventory item.
- Attach the item to a project room.
- Return to improve or combine items.

## Strategic Bet

The bet is not that people want another game.

The bet is that agent interaction becomes more memorable, differentiated, and commercially useful when agents, projects, workflows, and knowledge assets are embodied inside a world with persistent context, quests, inventory, and visible progress.

## Key Design Constraint

Every game mechanic must increase one of these:

- Useful output quality
- User retention
- Project clarity
- Community identity
- Monetizable trust
- Distribution leverage

If a mechanic does not help any of those, cut it from v1.
